# Скиллы

Скилл — это записанная инструкция, которую агент выполняет для повторяющейся задачи, чтобы каждый раз не объяснять заново. По сути — сохранённый промпт с протоколом. Роли (`ROLES.md`) — это тоже скиллы, просто самые крупные и постоянные.

**Как использовать.** Дай агенту прочитать нужный файл и скажи, например, «подготовь к встрече с X по скиллу meeting-prep» или «собери недельный дайджест». Агент следует протоколу.

**Как заводить свои.** Заметил, что раз за разом просишь одно и то же (собрать prep, свести отчёты, проверить свежесть статусов) — опиши это одним файлом: задача, когда запускать, протокол по шагам, что на выходе, чего не делать. Держи скилл коротким и конкретным.

В наборе для старта:

- `meeting-prep.md` — подготовка к встрече (шпаргалка твоего уровня).
- `weekly-digest.md` — недельный пульс направления на один экран.
- `self-mirror.md` — зеркало дня: ежедневный срез про твой управленческий стиль.
- `b24-bridge.md` — мост к порталу Битрикс24: подключение сенсора (вебхук), безопасность, свежие данные портала по запросу (календарь, задачи, оргструктура, люди, чаты).

# .b24-bridge — конфиг моста к Битрикс24

Здесь живёт настройка **сенсора** — связи агента с твоим порталом Битрикс24. Мост даёт агенту свежие данные портала (календарь, задачи, оргструктуру, карточки людей, чаты) по запросу.

## Что здесь

- `webhook.env.example` — шаблон конфига **без секретов**, под версионностью. Образец.
- `webhook.env` — твой рабочий файл с реальным URL вебхука. **Создаётся локально, в git не попадает** (в `.gitignore`). Появится после настройки.

## Как настроить

Полная инструкция — в `skills/b24-bridge.md`. Коротко:

```bash
cp .b24-bridge/webhook.env.example .b24-bridge/webhook.env
chmod 600 .b24-bridge/webhook.env
# затем впиши URL вебхука в webhook.env
```

## Безопасность

- URL вебхука — секрет уровня пароля. Живёт **только** в `webhook.env` с правами `600`. Никогда — в заметках, в чате с агентом, в git.
- Держи эту папку вне файл-синков (Obsidian Sync, облачные диски), чтобы токен не разъехался по устройствам.
- Скомпрометировал токен — выпусти новый вебхук на портале, старый удали, обнови только `webhook.env`.

Полная модель безопасности — в `skills/b24-bridge.md`, раздел «Безопасность».
